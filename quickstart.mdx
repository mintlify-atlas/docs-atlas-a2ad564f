## Run your first audit

Get started immediately with a free audit—no API key required.

<Steps>
  <Step title="Install Stobo CLI">
    If you haven't already, install Stobo CLI using pip:
    
    ```bash
    pip install stobo
    ```
    
    <Info>
      See the [installation guide](/installation) for alternative installation methods and troubleshooting.
    </Info>
  </Step>
  
  <Step title="Run a site audit">
    Audit an entire website with a single command:
    
    ```bash
    stobo audit site https://example.com
    ```
    
    This command runs:
    - 30+ SEO checks (meta tags, headings, structured data, performance)
    - 7 AEO checks (answer engine optimization)
    - Blog detection
    - Sitemap discovery
    
    <Tip>
      Site audits are completely free and require no authentication.
    </Tip>
  </Step>
  
  <Step title="Audit a specific article">
    For article-level analysis, use the `audit run` command:
    
    ```bash
    stobo audit run https://example.com/blog/my-post
    ```
    
    This performs both SEO and AEO checks on the specific URL.
    
    **Filter by check type:**
    
    <CodeGroup>
    ```bash SEO only
    stobo audit run https://example.com/blog/my-post --seo-only
    ```
    
    ```bash AEO only
    stobo audit run https://example.com/blog/my-post --aeo-only
    ```
    
    ```bash With target keyword
    stobo audit run https://example.com/blog/my-post --keyword "content optimization"
    ```
    </CodeGroup>
  </Step>
  
  <Step title="View your results">
    Stobo CLI displays rich, color-coded results directly in your terminal:
    
    - ✓ **Passed checks** in green
    - ✗ **Failed checks** in red with recommendations
    - **Categorized results** by SEO area (content, technical, meta, etc.)
    - **Actionable suggestions** for each issue
    
    <Info>
      Want machine-readable output? Add the `--json` flag to any command:
      
      ```bash
      stobo audit site https://example.com --json
      ```
    </Info>
  </Step>
</Steps>

## Understanding audit results

### Site audit output

A site audit provides:

- **SEO categories**: Content, technical, meta tags, structured data, performance
- **AEO checklist**: Answer engine optimization factors
- **Blog detection**: Identifies blog sections and content patterns
- **Sitemap discovery**: Finds and validates XML sitemaps
- **Recommendations**: Prioritized action items

### Article audit output

An article audit includes:

- **SEO score**: Overall optimization rating
- **Individual checks**: Title, meta description, headings, images, internal links
- **AEO analysis**: Answer potential, entity coverage, structured snippets
- **Keyword optimization**: Target keyword usage and density (if specified)

## Common commands

Here are essential commands to get you started:

<CodeGroup>
```bash Site audit
# Comprehensive website audit
stobo audit site https://example.com
```

```bash Article audit
# Single page/article audit
stobo audit run https://example.com/blog/post
```

```bash List audits
# View your recent audits
stobo audit list
```

```bash Get specific audit
# Retrieve audit by ID
stobo audit get <audit-id>
```

```bash Get by URL
# Get most recent audit for a URL
stobo audit get-url https://example.com/blog/post
```
</CodeGroup>

## Global options

You can use these flags with any command:

| Option | Description | Example |
|--------|-------------|----------|
| `--json` | Output raw JSON | `stobo audit site https://example.com --json` |
| `--api-key` | Override stored API key | `stobo --api-key sk_... audit site https://example.com` |
| `--base-url` | Override API base URL | `stobo --base-url https://api.example.com ...` |
| `--version` | Show version | `stobo --version` |
| `--help` | Show help | `stobo --help` |

<Note>
  You can also set the API key via the `STOBO_API_KEY` environment variable instead of using the `--api-key` flag.
</Note>

## Example workflow

Here's a typical workflow for auditing and improving your content:

```bash
# 1. Audit your site
stobo audit site https://yourblog.com

# 2. Audit a specific article
stobo audit run https://yourblog.com/blog/latest-post

# 3. Authenticate for paid features (if needed)
stobo auth login

# 4. Extract your brand voice
stobo tone extract https://yourblog.com/blog

# 5. Run full optimization on an article
stobo optimize run https://yourblog.com/blog/latest-post

# 6. Check your credit balance
stobo credits
```

You're now ready to start auditing and optimizing your content with Stobo CLI!
